# boilerplate-core

基于 webpack4.0 版本的前端项目脚手架核心配置逻辑

![Version](https://img.shields.io/badge/version-1.0.4-blue)

发布步骤:

1. 修改版本号
2. 在 README 中添加更新日志
3. 提交代码
4. 通过 `git tag vx.x.x` 添加标签并提交
5. 执行 `npm run release` 发布新包

## ✨ 特性

整合了以下配置文件的核心逻辑:

- babel
- eslint
- karma
- tsconfig
- webpack

提供可执行脚本:

- lint-staged: 对修改的文件进行代码校验
- release: 发布模块
